TextPro Annotation Tool

TextPro (http://textpro.fbk.eu/) is a set of NLP modules which perform Web page cleaning, tokenization, sentence detection, morphological analysis, pos-tagging, lemmatization, chunking and named-entity recognition.

The input given to TextPro should contain only pure text (hub text, see the CES guidelines for corpus encoding: http://www.cs.vassar.edu/CES/CES1-5.html#ToCOview).
If a text contains any markup, this should be removed (HTML files can be manage using -html option. All tags will be removed and the relevant text is kept as starting input text).
More generally, the Textpro tabular format is not meant to represent information about the structure of the text (e.g. titles, paragraphs, emphasis).

The output file contains:
  • the header lines: some meta information, such as the name of the file, the document's creation date, the author, etc.., can be coded in the first lines. These lines begin with the special character "# ", followed by the name of the field and then its value;
  • the annotation lines contain all information about each token of the input file using tabular format (the columns are separated by tabular space).

    The end of sentence is marked with an empty line. If a module does not have an output to give, it provides the label __NULL__.

    HTML CLEANER

    A specific module of TextPro, called CleanPro, removes mark-up tags and irrelevant text (i.e. words used as navigation menu, common header and footer, etc.) from HTML pages. Using the option -html the input HTML file will be cleaned. The relevant text is kept as input text of the following modules.

    COMPATIBLE MORPHOLOGICAL ANNOTATIONS

    Considering the line:
     bearing   VVG   bear   bear+v+gerund+pres bearing+n+sing bearing+adj+zero
    First of all "bear+v+gerund+pres bearing+n+sing bearing+adj+zero" is one column (separators are spaces, not tabs).

    When TextPro is called the +full_morpho value for the -c option (see the textpro -help) this column contains all the possible morphological analysis for the token.
    If you want to get only the morphological anlysis/es that is/are compatible with the selected POS tag, then use the +comp_morpho. Note that there can be more than one morpho analysis compatible with a certain PoS tag.

    NAMED-ENTITY RECOGNITION

    The named-entity tagger (EntityPro) has recently been trained for Italian on the EVALITA (http://evalita.fbk.eu/) development set (an Italian manually annotated corpus) and for English on the CONLL-2003 dataset. Named entities are represented in the IOB2 format. The data contains entities of four types: Person (PER), Organization (ORG), Location (LOC) and Geo-Political entity (GPE, but only for Italian).

    At the *current stage* of development, the EntityPro module:

    - only recognizes NAMED entities ("John"). It does not recognize other types of entity mentions ("he", "the guy");
    - recognizes the extension and the type of a restricted set of named entities (persons, organizations, locations). It does not normalize named entities so as to recognize, that, for instance, "S. Berlusconi" and "Silvio Berlusconi" are the same person.

    Output example: 
     
# FIELDS: token	pos	wnpos	lemma	entity
Both	DT0		both	O
Mary	NP0	n	mary	B-PER
B.	PUN		b.	I-PER
and	CJC		and	O
George	NP0	n	george	B-PER
W.	AV0	r	w.	I-PER
went	VVD	v	go	O
to	PRP		to	O
London	NP0	n	london	B-LOC
.	PUN		.	O
    The entity annotation (the fourth column) consists of two parts:
  • the IOB2 tag: “B” (for “begin”) denotes the first token of a Named Entity, I (for “inside”) is used for all other tokens in a Named Entity, and O (for “outside”) is used for all other words;
  • the Entity type tag: PER (for Person), ORG (for Organization), LOC (for Location) and GPE (for Geo- Political Entity).

    CHUNKER

    Chunking for Italian annotates with 2 categories (using IOB-2 format like for named entity annotation): B-NP, B-VX.
    The output of the chunker with the TextPro's output could be as following:
    # FIELDS: token	pos	chunk
A       E       O
Trento  SPN     B-NP
,       XPW     O
assaggi SP      B-NP
di      E       O
teatro  SS      B-NP
da      E       O
    For English there are a number of categories larger: B-ADJP, B-ADVP, B-CONJP, B-INTJ, B-LST, B-NP, B-PP, B-PRT, B-SBAR, B-UCP, B-VP

    Legend about the labels of the Penn Treebank Tags: 
    ADJP - Adjective Phrase.
ADVP - Adverb Phrase.
CONJP - Conjunction Phrase.
INTJ - Interjection. Corresponds approximately to the part-of-speech tag UH.
LST - List marker. Includes surrounding punctuation.
NP - Noun Phrase. 
PP - Prepositional Phrase.
PRT - Particle. Category for words that should be tagged RP. 
SBAR - Clause introduced by a (possibly empty) subordinating conjunction.
UCP - Unlike Coordinated Phrase. 
VP - Verb Phrase.

    KEYWORDS EXTRACTION

    In the header line "# KEYWORDS:" indicates the most important keywords of the document (in lower case). An example is the following: 
    # KEYWORDS: ahmed yassawi <44.53 6> vox populi <30 6> ermanno visintainer <29.93 5> centro studi <28.22 6> kazakhstan <17.92 
7> pensiero filosofico <12.98 2> presidente di turno <11.92 1> osce <10.47 4> almaz khamzayev <10.42 2> ...
    Each keyword is followed by "<NUM1 NUM2>". NUM1 indicates the relevance of keyword. NUM2 indicates the number of occurrences of the keyword in the text.
